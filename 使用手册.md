## 1. 概述

Eagle Watcher 是一个 macOS 菜单栏应用，它**自动监控你的 Downloads 文件夹**，每当有新文件出现时：

- 分析文件名，匹配知识库中的关键词，自动归入对应主题
- 文件名模糊时（如 `screenshot_2025-01-01.png`），调用 AI 视觉分析识别内容
- 匹配失败的文件进入**通用箱**，你可以在 HUD 面板中手动整理
- 整理操作为知识库积累学习，下次同类型文件自动归类

### 前置要求

- **macOS 11+**（Big Sur 或更高）
- **[Eagle](https://eagle.cool)** 设计素材管理工具（需启用开发者选项 API）
- **Python 3.9+**

## 2. 安装

### 2.1 通过 pip 安装

```bash
# 核心功能（纯 API / CLI 模式，可在非 macOS 机器上安装）
pip install eagle-watcher

# macOS 完整功能（菜单栏 + 原生文件监控 + 浮动面板）
pip install 'eagle-watcher[full]'
```

### 2.2 验证安装

```bash
eagle-watcher --help       # 应显示帮助信息
eagle-import --help        # 应显示帮助信息
eagle-server --help        # 应显示帮助信息
```

### 2.3 开发模式安装

```bash
git clone https://github.com/yiwenxu123/eagle-watcher.git
cd eagle-watcher
pip install -e ".[full,test]"
```

## 3. 配置

### 3.1 首次运行

```bash
eagle-watcher
```

首次运行会自动创建配置文件 `~/.eagle-watcher/config.yaml`，并提示你完成 Eagle API Token 配置。

### 3.2 配置 Eagle API

1. 打开 **Eagle**
2. 进入 **Eagle → 设置 → 开发者选项**
3. 复制显示的 **API Token**

编辑配置文件：

```yaml
# ~/.eagle-watcher/config.yaml
eagle:
  host: "http://localhost:41595"   # Eagle 默认端口，一般不需要改
  token: "你的API Token"            # 粘贴到这里

paths:
  downloads: "/Users/你的用户名/Downloads"  # 监控的下载目录
  watch_interval: 2.0                       # 轮询间隔（秒）
  extra_watch_dirs: []                       # 额外监控目录（可选）

notifications:
  inbox_reminder: true    # 通用箱有新素材时通知
  import_success: false   # 导入成功时通知

ai:
  api_key: ""             # DashScope API Key（AI 视觉分析用）

delete_after_import: "trash"  # 导入后处理：keep（保留）/ trash（废纸篓）
```
### 3.3 AI API Key（可选，但推荐）

AI 视觉分析使用阿里云通义千问（Qwen-VL）。

方式一：写入配置文件
```yaml
ai:
  api_key: "sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

方式二：设置环境变量
```bash
export DASHSCOPE_API_KEY="sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

> 配置文件中设置优于环境变量。不配置 AI Key 则 AI 视觉分析不可用，模糊文件名的素材直接进入通用箱。

### 3.4 配置校验

```bash
# 启动时自动校验，你也可以手动查看配置状态：
eagle-watcher
# 启动日志中会显示配置错误/警告
```

## 4. GUI 模式（菜单栏 + 面板）

### 4.1 启动

```bash
eagle-watcher
```

启动后，菜单栏出现图标（显示当前项目名称或"自动匹配"）。

### 4.2 菜单栏

点击菜单栏图标：

| 菜单项 | 说明 |
|--------|------|
| 🪟 打开面板 | 打开/切换 HUD 浮动面板 |
| 📂 监控目录 | 显示当前监控的目录及状态（✅ 正常/❌ 不存在） |
| 🚪 退出 | 退出应用 |

### 4.3 HUD 浮动面板

面板是主要操作界面，位于屏幕右上角。

#### 状态区

| 信息 | 说明 |
|------|------|
| 今日入库 | 当天已导入的素材数量 |
| 通用箱 | 待分类素材数量 |
| Eagle 状态 | 在线/离线 |
| AI 状态 | 已配置/未配置（显示 AI 模型名） |
| 权限状态 | 是否有 Downloads 目录写入权限 |

#### 主题/项目管理

- **切换当前主题**：点击主题名称，后续文件自动归入该主题
- **创建主题**：输入名称 → 选择分类 → 添加默认标签
- **删除主题**：自动删除关联的分类文件夹
- **主题与分类**：主题（project）挂靠在分类（category）下，分类对应 Eagle 中的文件夹

#### 通用箱整理

通用箱中是文件名匹配失败的素材。每个素材显示：

- 缩略图
- AI 建议的主题（基于文件名二次匹配）
- 置信度

操作：

| 操作 | 说明 |
|------|------|
| **确认** | 选择主题和标签 → 确认整理 → 知识库自动学习 |
| **教AI认识它** | 手动为素材指定主题和标签，AI 学会下次自动归类 |
| **跳过** | 移除"待分类"标签，添加"已跳过"标签 |

> 整理后的素材标签已更新，但文件不会移动到不同 Eagle 文件夹——Eagle API 不支持移动操作。

#### 临时监控目录

除了永久的 Downloads 目录，你还可以添加**临时目录**进行监控：

1. 点击 **"添加临时目录"** → 弹出系统文件夹选择器
2. 选中一个文件夹 → 添加到监控列表
3. 可以对临时目录执行**批量扫描**：选择过滤类型（图片/文档/视频）→ 点击扫描
4. 扫描前会预览文件数量和类型分布

临时目录仅在当前运行期有效，重启后不会持久化。

### 4.4 快捷键

| 快捷键 | 说明 |
|--------|------|
| `Cmd + W` | 关闭 HUD 面板 |
| 面板拖拽把手 | 拖动面板到任意位置 |
| 右下角拖拽 | 调整面板大小（360-800 × 400-1200） |

## 5. CLI 命令行

### 5.1 单次导入

```bash
# 导入本地文件到指定主题
eagle-import --file "白起.jpg" --project "武安侯" --tags "战国,武将"

# 从 URL 导入
eagle-import --url "https://example.com/poster.jpg" --project "海报参考"

# 不指定主题——自动文件名匹配
eagle-import --file "white_qi.jpg"
# 匹配知识库后自动归类；匹配失败则进通用箱

# 指定 Eagle 文件夹（覆盖主题的默认文件夹）
eagle-import --file "design.ai" --project "设计素材" --folder "设计库/AI资源"
```

### 5.2 启动目录监控

```bash
# 监控 Downloads + 额外指定目录
eagle-import --watch-dir ~/Pictures --watch-dir ~/设计素材
# 按 Ctrl+C 停止
```

### 5.3 批量扫描目录

在面板中添加临时目录后，可以执行批量扫描。扫描前会：

1. 显示文件类型分布预览
2. 选择过滤预设（全部/图片/文档/视频）
3. 扫描过程中实时显示进度
4. 扫描完成显示统计结果

## 6. HTTP API

HTTP API 供远程 AI 代理（如 Hermes、OpenClaw）通过 SSH 隧道调用。

### 6.1 启动服务器

```bash
# 纯 API 模式（无 GUI 依赖）
eagle-server
# 输出：🌐 HTTP Server 已启动：http://127.0.0.1:9800
```

远程代理通过 SSH 隧道连接你的 Mac：
```bash
ssh -L 9800:localhost:9800 your-mac
```

### 6.2 端点

#### 健康检查

```bash
curl http://localhost:9800/ping
# {"status": "ok", "eagle_online": true}
```

#### 查询状态

```bash
curl http://localhost:9800/status
# {"status": "ok", "data": {"project": "武安侯", "projects": [...], "eagle_online": true, "folders": [...]}}
```

#### 导入素材

```bash
# 从 URL 导入
curl -X POST http://localhost:9800/import \
  -H "Content-Type: application/json" \
  -d '{"file_url": "https://example.com/poster.jpg", "project": "海报参考"}'

# 从本地路径导入
curl -X POST http://localhost:9800/import \
  -H "Content-Type: application/json" \
  -d '{"file_path": "/Users/name/Downloads/image.jpg", "tags": ["参考"]}'

# 不指定 project——自动通过 decide() 决策引擎匹配
curl -X POST http://localhost:9800/import \
  -H "Content-Type: application/json" \
  -d '{"file_path": "/Users/name/Downloads/白起剧照.jpg"}'
```

**响应格式**：
```json
{"status": "success", "message": "已入库：海报参考", "data": {"tags": [...], "folder": "海报参考"}}
```

## 7. AI 视觉分析

### 7.1 触发条件

AI 视觉分析仅在以下条件**全部满足**时触发：

1. 文件是图片格式（jpg/png/gif/webp/bmp/svg 等）
2. 文件名被认为"模糊"：
   - 纯数字文件名（≥5 位）
   - 32 位十六进制哈希
   - 纯字母 ≥20 位 / 纯字母数字 ≥30 位
   - 短随机串（含数字的 5-15 位字母数字组合）
   - **中英文截图名**：包含「截图」「截屏」「屏幕快照」「微信图片」「mmexport」「IMG_」「wx_camera」等关键字的文件名
3. 文件大小 ≤ 20MB
4. 已配置 DashScope API Key
5. 没有已缓存的分析结果

### 7.2 AI 识别内容

AI 分析图片后返回：
- **3-5 个中文标签**：描述图片内容（如"人物","古装","白起","战国"）
- **文件名建议**：简短描述（5-15 字）

### 7.3 缓存机制

AI 分析结果按图片 MD5 哈希缓存 30 天。相同图片再次出现时直接使用缓存，不重复调用 API。

缓存路径：`~/.eagle-watcher/cache/`

### 7.4 知识库学习

AI 分析成功后，若当前设定了项目主题，AI 返回的第一个**非通用标签**（排除 portrait、landscape、screenshot 等通用英文词）会被学习到知识库中。下次同名文件出现时直接匹配，无需再次调用 AI。

## 8. 知识库系统

### 8.1 工作原理

知识库存储在 `~/.eagle-watcher/knowledge.yaml`，结构如下：

```yaml
keywords_mapping:
  白起:
    theme: "武安侯"
    tags: ["战国", "武将"]
    confidence: 0.85
    match_count: 2
    first_seen: "2026-01-15"
    source: "user_confirmed"
  qin:
    theme: "秦始皇"
    tags: ["秦朝", "文物"]
    confidence: 0.70
    match_count: 1
    source: "user_confirmed"

sources:
  pinterest.com:
    theme: "设计灵感"
    tags: ["设计", "参考"]

misses: []
```

### 8.2 匹配优先级

文件名匹配知识库时，按以下优先级：

1. **完整词匹配**（最高）：文件名分词后，某个词完全等于关键词
2. **词边界匹配**：关键词在文件名中以词边界出现（仅对 ASCII 关键词有效）
3. **部分匹配**（最低）：关键词是文件名某个词的子串（置信度 ×0.8）

### 8.3 置信度

- 新建关键词：`0.70`
- 每次成功匹配增加：`+0.15`
- 最高上限：`0.98`
- 清理阈值：低于 `0.30` 的条目被删除
- 1 次匹配后超过 90 天未使用 → 自动清理

### 8.4 学习方式

知识库通过以下方式自动学习：

| 触发场景 | 学习内容 |
|---------|---------|
| 通用箱整理确认 | 文件名关键词 → 所选主题 |
| AI 分析成功（有当前主题） | 第一个非通用 AI 标签 → 当前主题 |
| HTTP API 导入（无 project 参数） | 文件名关键词 → 决策引擎匹配的主题 |
| 创建项目 | 项目名 → 所属分类 |

## 9. 常见问题

### 9.1 Eagle 未运行

如果启动时 Eagle 未运行，应用会：
1. 在菜单栏显示 Eagle 离线状态
2. 后台每 30 秒尝试重连
3. Eagle 上线后自动启动文件监控
4. 按 `Cmd + W` 关闭面板后可随时通过菜单栏重新打开

### 9.2 文件未被监控

检查以下几点：
- Downloads 目录存在且有读取权限
- 系统偏好设置 → 隐私与安全性 → 文件和文件夹 → 允许终端/应用访问
- 配置中的路径是否正确（`os.path.expanduser` 自动展开 `~`）

### 9.3 AI 分析不工作

- 确认已配置 `ai.api_key` 或环境变量 `DASHSCOPE_API_KEY`
- 仅图片格式（jpg/png/gif/webp/bmp/svg）触发 AI
- 文件大小 ≤ 20MB
- 文件名模糊条件见[第 7.1 节](#71-触发条件)

### 9.4 如何清除 AI 缓存

面板中操作：系统 → 清除 AI 缓存

或手动删除：
```bash
rm -rf ~/.eagle-watcher/cache
```

### 9.5 日志在哪

```bash
# 查看今日日志
cat ~/.eagle-watcher/log/eagle-watcher_20260602.log

# 查看实时日志
tail -f ~/.eagle-watcher/log/eagle-watcher_*.log
```

### 9.6 如何重置全部数据

```bash
# 删除所有配置和数据（谨慎操作！）
rm -rf ~/.eagle-watcher
```

### 9.7 macOS 权限问题

如果提示"没有权限访问 Downloads"，请：

1. 打开 **系统设置 → 隐私与安全性 → 文件和文件夹**
2. 找到你的终端应用（或 eagle-watcher）
3. 开启 **"Downloads 文件夹"** 的访问权限
4. 如果找不到，尝试在面板中点击 **"打开隐私设置"** 按钮

### 9.8 面板不显示/闪退

可能是缺少 GUI 依赖，确认安装完整版本：
```bash
pip install 'eagle-watcher[full]'
```

如果 PyObjC 安装失败，可以改用 HTTP API 模式：
```bash
eagle-server
```
