## 一、产品定位

一个 Mac 本地常驻的"素材管家"，实现：

| 场景 | 你的操作 |
|------|---------|
| **手动下载素材**（浏览器保存） | 设置/切换主题 → 后续自动入库 |
| **没有主题随手存** | 自动识别已有主题或进通用箱，不弹窗 |
| **Agent 自动下载**（Hermes/OpenCode/Claude Code/HanaAgent/OpenClaw） | 零操作 |

### 远程 Agent 调用方式

远程 Agent（Hermes、OpenClaw）通过 Mac 上额外跑的一个轻量 HTTP 服务调用：

```
Hermes (腾讯云) ──→ POST http://mac:9800/import
                     {"file_url": "...", "project": "武安侯", "tags": ["白起"]}
                         │
                         ▼
                   HTTP Server (mac 本机)
                         │
                         ▼
                   Eagle API (localhost:41595) → 入库

OpenClaw (Windows) ──→ 同样经过 HTTP → 入库
```

CLI 和 HTTP Server 共用同一套 `eagle_api.py` 核心模块，功能完全一致。

## 三、主题决策引擎（核心逻辑）

### 3.1 优先级规则

```
新文件出现
    ↓
┌─ 是否设了当前主题？（state.json）─────────────────────┐
│  ✅ 是 → 强制归入当前主题                              │
│         跳过所有 AI 匹配，直接入库                      │
│                                                       │
│  ❌ 否 → 进入 AI 匹配流程                              │
│          ↓                                            │
│     文件名/来源 是否匹配到已有主题？                     │
│          ↓                                            │
│      ✅ 匹配 → 归入该主题                               │
│      ❌ 不匹配 → 进通用箱                               │
└───────────────────────────────────────────────────────┘
```

**关键约束**：
- AI **只能匹配已有主题**，不能自动创建新主题
- 用户手动设了主题 = 最高优先级，AI 不干预
- 取消主题 → 恢复默认（AI 匹配或进通用箱）

### 3.2 主题管理

主题由用户创建，AI 只匹配不创建：

```yaml
# themes.yaml（用户创建，或 Hermes 协助创建）
themes:
  武安侯:
    created_at: "2026-05-22"
    default_tags: ["战国", "历史人物"]
    eagle_folder: "武安侯"
  秦始皇:
    default_tags: ["秦朝", "皇帝"]
    eagle_folder: "秦始皇"
  海报参考:
    default_tags: ["设计", "排版"]
    eagle_folder: "海报参考"
```

主题创建途径（仅限用户或 Hermes 指令）：
1. 在菜单栏手动创建
2. Hermes 启动项目时创建（"开始武安侯项目"）
3. 整理通用箱时创建

**AI 在任何情况下不允许自动创建主题。**

### 3.3 知识库自动积累

```yaml
# knowledge.yaml（AI 自动积累，跨设备同步）
keywords_mapping:
  白起:
    theme: 武安侯
    confidence: 0.95
    tags: [白起, 战国, 秦, 武将]
    first_seen: "2026-05-22"
    match_count: 3
    source: user_confirmed    # 用户确认过的
  
  兵马俑:
    theme: 秦始皇
    confidence: 0.9
    tags: [兵马俑, 秦朝, 秦始皇]
    match_count: 2
    source: user_confirmed

  武王:
    theme: null              # 未归类
    confidence: 0.6
    tags: [周朝]
    match_count: 1
    source: ai_suggested      # AI 建议但未确认

  # 来源网站映射
  sources:
    baidu.com/战国:
      theme: 武安侯
      tags: [战国, 百度搜索]
```

**学习机制**：

```
用户第一次下载"白起剧照.jpg"
  → 关键词"白起"无匹配 → 进通用箱
  → 用户手动归入"武安侯" → 系统记录映射
  → knowledge.yaml 新增：白起 ↔ 武安侯（confidence: 0.7）

用户第二次下载"白起肖像.png"
  → 匹配到知识库 → 自动归入武安侯
  → match_count: 2 → confidence 升至 0.85

用户第三次下载"白起.gif"
  → 高置信度匹配 → 自动归入
  → match_count: 3 → confidence 升至 0.95
```

## 四、Eagle 素材管理最佳实践

### Eagle 双维管理模型

Eagle 的核心机制是 **文件夹 + 标签** 双维度，正好适合你的场景：

```
文件夹（1个素材只能属于1个文件夹）
  → 适合"大分类"——每个项目/主题一个文件夹
  → 工作时直接打开项目文件夹，所有素材都在里面

标签（1个素材可以有无数个标签）
  → 适合"细节维度"——跨文件夹搜索
  → 搜"白起"不管在哪个主题都能找到
```

### 推荐结构

```
图片素材.library
├── 📂 武安侯              ← 一个项目/主题一个文件夹
│   ├── 白起剧照.jpg       ← 标签：白起, 战国, 武将, 人物画像, 搜索引擎
│   └── 苏秦佩六国相印.jpg  ← 标签：苏秦, 战国, 纵横家, 人物画像, 搜索引擎
├── 📂 秦始皇
│   ├── 兵马俑.jpg         ← 标签：兵马俑, 秦朝, 文物, 搜索引擎
│   └── 长城地图.png       ← 标签：长城, 秦朝, 地图, AI生成
├── 📂 海报参考
│   └── 配色参考.jpg       ← 标签：设计, 配色, 灵感, 海报
└── 📂 _通用箱              ← 未分类素材暂存
    └── 未分类.jpg          ← 标签：待分类
```

**为什么这样好**：
- 你做"武安侯"视频时 → 直接打开武安侯文件夹，所有素材一览无余
- 以后做其他视频需要"白起"素材 → 搜索"白起"标签，跨主题也能找到
- 标签还能组合搜索：`白起 + 画像` 精确找到你要的

## 五、标签系统（AI 的工作重点）

### 5.1 四维标签体系

每条素材入库时生成四类标签：

| 标签维度 | 来源 | 示例 |
|---------|------|------|
| 🏷 **主题标签** | 当前主题 / AI 匹配 | `武安侯` `秦始皇` |
| 📖 **内容标签** | 文件名 + 知识库 + AI 视觉 | `白起` `战国` `兵马俑` |
| 🎨 **类型标签** | AI 分析图片内容 | `人物画像` `场景` `文物` `地图` |
| 📎 **来源标签** | 下载来源 / Agent 标注 | `搜索引擎` `AI生成` `电视剧截图` |

### 5.2 AI 打标签策略（分层方案）

```
文件名"武王伐纣.jpg"
    ↓
① 文件名拆词：武王、伐纣
    ↓
② 知识库匹配：
   └─ "武王" → 置信度 0.9 → 标签：周朝, 西周, 武王
    ↓
③ 类型推断（按需调用 Qwen-VL）：
   └─ 这是一幅古代绘画 → 标签：古代绘画, 历史题材
    ↓
④ 来源判断：
   └─ 来自 baidu.com → 标签：搜索引擎
    ↓
最终入库标签：[周朝, 西周, 武王, 伐纣, 古代绘画, 历史题材, 搜索引擎]
```

### 5.3 AI 视觉分析方案

| 要点 | 决策 |
|------|------|
| **使用哪个模型** | **Qwen-VL**（阿里通义千问），通过 DashScope API 调用 |
| **你的 API Key** | ✅ 已配置在 `~/.zshrc` 中（`DASHSCOPE_API_KEY`）|
| **调用时机** | 仅在 L0（文件名）+ L1（来源）都无法判断时才调用 |
| **预估频率** | 每周约 10-20% 的素材需要，即 ≈ 6-12 张/周 |
| **预估成本** | Qwen-VL ≈ ¥0.004/张 → 每周 ≈ ¥0.03-0.05，几乎免费 |
| **为什么不用 DeepSeek** | DeepSeek V4 目前是纯文本模型，不支持图片理解 |

**为什么不所有图片都用 AI 看**：

你的素材中约 60-70% 文件名本身就包含了足够信息（如"白起剧照.jpg"），不需要浪费 API 调用。只有完全看不出内容的文件（如"下载.jpg"）才需要 AI 理解。

### 5.4 标签来源优先级

```
手动指定 > 主题默认标签 > 知识库匹配 > AI 视觉分析
```

### 5.5 标签一致性保障

AI 每次打标签前先查 Eagle 已有标签库，**复用已有标签**，杜绝同义标签膨胀：

```
Eagle 已有标签：["战国", "白起", "武将"]
  ↓
AI 想添加：["战国时期", "白起将军"]
  ↓
AI 校验发现已有"战国"和"白起"
  ↓
实际入库：["战国", "白起"]  ← 复用已有标签
```

### 5.6 视频素材处理原则

视频素材仅使用 L0（文件名）+ L1（来源）分析，**不做 AI 视觉理解**。原因：
- 视频分析成本高（逐帧 / 关键帧提取）
- 你目前的素材以图片为主，视频少量

## 六、多 Agent 兼容设计

### 6.1 CLI 模式（本地 Agent 用）

```bash
# OpenCode / Claude Code / HanaAgent 直接执行
eagle-import --file "白起.jpg" --project "武安侯" --tags "白起,战国"

# 远程 URL 导入
eagle-import --url "https://example.com/兵马俑.jpg" --project "秦始皇"
```

本地 Agent 直接调 CLI，像执行普通命令一样。

### 6.2 HTTP 模式（远程 Agent 用）

Mac 上额外运行一个轻量 HTTP 服务（与 CLI 共用 `eagle_api.py` 核心模块）：

```
Hermes (腾讯云)     → POST http://mac:9800/import
OpenClaw (Windows) → POST http://mac:9800/import
                         │
                         ▼
                    eagle-watcher HTTP Server
                    读取请求参数 → 调用核心模块
                         │
                         ▼
                    Eagle API (localhost:41595)
```

**对远程 Agent 来说，它们只需要能发出 HTTP 请求（这是任何 Agent 最基本的能力），不需要关心 Mac 端的实现细节。**

## 七、跨设备可移植设计

### 7.1 数据分层

```
可同步的数据（有价值，跨设备共享）
├── knowledge.yaml    ← 知识库，辛苦积累的核心资产
├── themes.yaml       ← 主题列表
├── state.json        ← 当前主题状态（可跨设备）

不可同步的数据（设备特有）
├── config.yaml       ← 本机路径、端口等配置
├── Eagle 本身        ← 各 Mac 各自安装
├── log/              ← 日志（不跨设备）
```

### 7.2 换设备的操作流程

```
新 Mac 上：
① 安装 Eagle（官网下载）
② 安装 Python 依赖
③ 从旧 Mac 拷 ~/.eagle-watcher/ 过来
   或 Syncthing 自动同步
④ 修改 config.yaml 里的 Downloads 路径（如有差异）

→ 知识库 knowledge.yaml 已经同步过来了，
  以前积累的匹配关系在新设备上直接用
  不需要重新训练
```

### 7.3 关于移动端（iPad/iPhone）

Eagle 没有移动端 App，所以移动端只能通过网页浏览器 + Eagle 的网页收藏功能保存素材。这套工具专注于 Mac 端的自动化管理。

## 十、技术选型

| 需求 | 选型 | 理由 |
|------|------|------|
| 语言 | Python 3.12 | Homebrew 已装 |
| 常驻后台 | Python + `launchd` | macOS 原生进程管理，开机自启 |
| Downloads 监控 | `macos.FSEvents`（内置） | 不依赖外部工具 |
| 菜单栏 UI | `rumps` | 纯 Python，macOS 原生体验 |
| Eagle API | `httpx` / `urllib` | 纯 HTTP，不需要 SDK |
| AI 视觉 | `dashscope` SDK（Qwen-VL） | 已配 API Key，中文理解强 |
| 通知 | `osascript` / `pyobjc` | macOS 原生通知 |
| HTTP 服务 | `http.server`（内置）或 `flask` | 轻量，依赖少 |
| 包管理 | `uv` 或 `pip` | 你已安装 |

| # | 模块 | 功能 | 说明 | 代码量 |
|:-:|:----:|------|------|:------:|

| 2 | `cli.py` | 命令行入口 | `eagle-import` 命令，Agent 直接调用 | 50 行 |

| 4 | `watcher.py` | 文件夹监控 | FSEvents 监听 Downloads | 60 行 |
| 5 | 决策核心 | 主题匹配逻辑 | 设主题→强制归入；无主题→AI匹配/通用箱 | 40 行 |

- 手动设主题 → 下载自动入库 Eagle + 正确标签
- Agent 调用 `eagle-import` → 直接入库
- 无主题 → 匹配已有主题或进通用箱
- 知识库自动学习（第二次同类文件不再进通用箱）

## 十二、与 Agent 的集成

### 12.1 本地 Agent（OpenCode / Claude Code / HanaAgent）

### 12.2 远程 Agent（Hermes 腾讯云 / OpenClaw Windows）

```
Hermes SSH 到 Mac → 调 CLI 或 HTTP
方案 A（推荐）：开启 Mac SSH → Hermes 直接 ssh 执行 eagle-import

```

```bash
# 统一调用格式（CLI 和 HTTP 共用）

  --tags "白起,战国,武将" \          # 标签（可选）
  --folder "人物"                   # Eagle 子文件夹（可选）
```
